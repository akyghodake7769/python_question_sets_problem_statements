================================================================================
AWS_WIN_01_M: Windows Server Basics - STUDENT IMPLEMENTATION & TESTING GUIDE
================================================================================

🎯 GOAL: Windows Server Basics: Navigation, Environment Variables & File Operations

This guide details the exact steps a learner must perform locally on the Windows VM.

--------------------------------------------------------------------------------
STEP-BY-STEP IMPLEMENTATION
--------------------------------------------------------------------------------

STEP 1: LOCAL ENVIRONMENT VERIFICATION (PASSES TC1)
- Log into the local Windows VM.
- Ensure that the Windows OS environment is active and running.

STEP 2: CONFIGURE DIRECTORY LAYOUT (PASSES TC2 & TC3)
- Open PowerShell as Administrator.
- Create the logs directory (C:\workspace\logs):
  New-Item -ItemType Directory -Path "C:\workspace\logs" -Force
- Create the backups directory (C:\workspace\backups):
  New-Item -ItemType Directory -Path "C:\workspace\backups" -Force

STEP 3: SET UP SYSTEM ENVIRONMENT VARIABLES (PASSES TC4)
- In the PowerShell session, configure the system-level variable permanently to 'production':
  [Environment]::SetEnvironmentVariable("APP_ENVIRONMENT", "production", "Machine")

STEP 4: SYSTEM METADATA AUDITING (PASSES TC5)
- Retrieve the hostname and OS caption details, and write them to C:\workspace\sysinfo.txt:
  $hostname = hostname
  $os = (Get-CimInstance Win32_OperatingSystem).Caption
  "Hostname: $hostname`nOS: $os" | Out-File -FilePath "C:\workspace\sysinfo.txt" -Encoding ascii -Force

STEP 5: LOG AUDITING (PASSES TC6)
- Locate all .log files directly inside the C:\Windows folder (non-recursively) and save their absolute paths to C:\workspace\log_files.txt:
  Get-ChildItem -Path "C:\Windows" -Filter "*.log" -File | Select-Object -ExpandProperty FullName | Out-File -FilePath "C:\workspace\log_files.txt" -Encoding ascii -Force

--------------------------------------------------------------------------------
HOW TO VERIFY LOCALLY
--------------------------------------------------------------------------------
- Switch to the student workspace terminal.
- Run local evaluation:
  python student_workspace/run.py
